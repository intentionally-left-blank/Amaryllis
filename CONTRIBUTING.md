# How To Contribute

Thanks for contributing to Amaryllis.

## Workflow

1. Open an issue describing the change and acceptance criteria.
2. Create a branch from `main`.
3. Keep pull requests focused and small.
4. Add or update tests for behavior changes.
5. Run release gates locally when relevant.

## Commit Requirements

- Every commit must include a `Signed-off-by:` trailer.
- We enforce DCO for all contributions.

Example:

```text
Signed-off-by: Your Name <you@example.com>
```

## Pull Requests

- Link issue(s) in PR description.
- Include risk/rollback notes for runtime or release changes.
- Do not bypass security or policy gates.
